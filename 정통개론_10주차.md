## 1. 통신망의 큰 그림

통신망을 **속도와 범위**로 나누면 크게 3가지로 볼 수 있어요:

| 구분 | 속도 특징 | 대표 기술 |
|------|-----------|-----------|
| **광역 통신망 (WAN)** | 기본 속도 | 인터넷 백본, 전화망 |
| **광대역 융합망 (MAN 포함)** | 고속 | 도시 간 네트워크 |
| **초고속 유선 가입자망** | 초고속 | 우리 집까지 들어오는 인터넷 |

### 🔍 근거리 통신망 (LAN) vs 광역 통신망 (WAN) 비교

**LAN (Local Area Network)**
- **범위**: 건물 내부, 캠퍼스 등 좁은 지역
- **계층 구조**: 
  - **LLC (Logical Link Control)**: 논리적 연결 제어 - 상위 계층과의 인터페이스
  - **MAC (Medium Access Control)**: 매체 접근 제어 - 물리적 주소(MAC 주소) 관리
- **OSI 계층**: 주로 1~2계층 (물리, 데이터링크)

**WAN (Wide Area Network)**
- **범위**: 도시, 국가, 대륙을 연결하는 광범위한 네트워크
- **OSI 계층**: **1~3계층까지 커버** (물리, 데이터링크, 네트워크)
  - 3계층(네트워크 계층)에서 **라우팅** 기능 수행
  - IP 주소 기반 경로 선택

> 💡 **핵심 차이점**: LAN은 가까운 거리라 간단한 주소(MAC)만 필요하지만, WAN은 먼 거리를 연결하므로 경로 찾기(라우팅)가 필수!

## 2. 광역 통신망 (WAN)

### 2.1 교환 방식의 분류

WAN의 교환 방식은 크게 **2가지**로 나뉩니다:

```
교환 방식
├── 회선 교환 (Circuit Switching)
│   ├── 공간 분할 교환 (Space Division)
│   └── 시간 분할 교환 (Time Division)
│
└── 축적 교환 (Store-and-Forward Switching)
    ├── 메시지 교환 (Message Switching)
    ├── 패킷 교환 (Packet Switching)
    │   ├── 가상 회선 방식 (Virtual Circuit)
    │   └── 데이터그램 방식 (Datagram)
    └── 셀 교환 (Cell Switching)
```

---

### 2.2 회선 교환 (Circuit Switching)

**개념**: 통화 시작부터 끝까지 **전용 회선을 독점**하는 방식

> 🎯 **비유**: 전화 통화할 때처럼, 통화가 시작되면 나만의 전용 선이 연결되는 것!

#### 📌 공간 분할 교환 (Space Division Switching)
- **물리적으로 다른 경로**를 할당
- 예: 전화국의 물리적 스위치가 직접 회선을 연결

#### 📌 시간 분할 교환 (Time Division Switching)
- **시간 슬롯**을 나눠서 여러 통신 처리
- 예: 1초를 1000조각으로 나눠서 각 통화에 순서대로 할당

**장점**: 
- 전용 회선이라 **지연 시간이 일정**함
- 실시간 통신에 유리 (전화 통화)

**단점**:
- 회선을 독점하므로 **효율이 낮음** (통화 안 해도 회선 점유)
- 초기 연결 설정 시간 필요

---

### 2.3 축적 교환 (Store-and-Forward)

**개념**: 데이터를 **일단 저장**했다가 전송하는 방식

> ⚠️ **중요**: 축적 교환은 **실시간 전송이 되지 않음**!

#### 📌 메시지 교환 (Message Switching)
- **메시지 전체**를 저장했다가 전송
- 메시지가 클 경우 지연 시간이 매우 김
- 현재는 거의 사용 안 함

#### 📌 패킷 교환 (Packet Switching) ⭐

**핵심**: 메시지를 **작은 패킷**으로 나눠서 전송

> 💡 **비유**: 큰 택배 상자를 작은 상자 여러 개로 나눠서 보내는 것!

**패킷의 구조**:
```
[ 헤더 (Header) | 데이터 (Data) ]
```
- **헤더**: 반드시 **주소 정보** 포함!
- **데이터**: 실제 전송할 내용

##### 🔸 가상 회선 방식 (Virtual Circuit)

**특징**:
- 통신 **시작할 때 경로를 미리 설정**
- 모든 패킷이 **같은 경로**로 전송
- 순서대로 도착함

**과정**:
1. **연결 설정** (Call Setup): 경로 결정
2. **데이터 전송**: 정해진 경로로 전송
3. **연결 해제** (Disconnect)

> 🎯 **비유**: 기차 철로처럼, 출발 전에 경로를 정하고 그 레일만 따라가는 것!

**장점**:
- 순서가 보장됨
- 헤더가 간단함 (경로 정보 필요 없음)

**단점**:
- 초기 연결 설정 시간 필요
- 경로 상의 한 노드가 고장나면 전체 통신 불가

##### 🔸 데이터그램 방식 (Datagram)

**특징**:
- **경로를 미리 정하지 않음**
- 각 패킷이 **독립적으로** 전송
- 패킷마다 **다른 경로**로 갈 수 있음
- 각 패킷에 **순서 정보** 포함

> 🎯 **비유**: 택배 여러 개를 보내는데, 각 택배가 알아서 빠른 길 찾아가는 것! (어떤 건 고속도로, 어떤 건 국도)

**장점**:
- 연결 설정 불필요 → 빠른 시작
- 한 경로가 막혀도 다른 경로 이용 가능 (유연함)

**단점**:
- 순서가 뒤바뀔 수 있음 (수신 측에서 재정렬 필요)
- 헤더가 복잡함 (매번 전체 주소 정보 필요)

##### 🔸 패킷 교환의 장단점

**장점** ✅
- 회선 효율이 높음 (여러 사용자가 공유)
- 패킷 단위로 오류 제어 가능
- 네트워크 자원을 효율적으로 사용
- 서로 다른 속도의 장치 간 통신 가능

**단점** ❌
- 패킷 처리 오버헤드 (헤더 추가, 재조립 등)
- 실시간 서비스에 부적합 (지연 시간 변동)
- 혼잡 시 지연 증가
- 각 패킷마다 라우팅 필요 (데이터그램의 경우)

#### 📌 셀 교환 (Cell Switching)
- **고정 크기의 작은 패킷**(셀)을 사용
- ATM에서 사용 (53바이트 셀)

---

### 2.4 다중화 (Multiplexing)

**개념**: 하나의 통신 회선을 **여러 사용자가 공유**하는 기술

#### 📌 TDM (Time Division Multiplexing) - 시간 분할 다중화
- **시간**을 나눠서 사용
- 각 사용자에게 **시간 슬롯** 할당

```
시간 →
[사용자A][사용자B][사용자C][사용자A][사용자B][사용자C]...
```

#### 📌 FDM (Frequency Division Multiplexing) - 주파수 분할 다중화
- **주파수 대역**을 나눠서 사용
- 각 사용자에게 **다른 주파수** 할당

```
주파수 ↑
사용자C ━━━━━━━━━━━
사용자B ━━━━━━━━━━━
사용자A ━━━━━━━━━━━
        시간 →
```

> 💡 **시험 꿀팁**: 라디오 방송이 FDM! (각 방송국마다 다른 주파수)

---

### 2.5 WAN 서비스 종류

#### 📌 PSTN (Public Switched Telephone Network)
- **공중 교환 전화망**
- 우리가 쓰는 일반 전화망
- **VoIP** (Voice over IP): 인터넷으로 전화 거는 기술
  - 💡 **모뎀 필요**: 디지털 ↔ 아날로그 변환

#### 📌 PSDN (Packet Switched Data Network)
- **패킷 교환 공중 데이터 통신망**
- 데이터 전송 전용 네트워크

##### X.25 프로토콜
- PSDN의 **표준 프로토콜**
- 가상 회선 방식 사용
- 오류 제어 기능 강력 (신뢰성 높음)

#### 📌 N-ISDN (Narrowband ISDN)
**협대역 종합 정보 통신망**

- 음성, 데이터, 영상을 **하나의 네트워크**로 통합
- **NT (Network Termination)**: ISDN 전용 모뎀 역할

**서비스 종류**:
- **BRI (Basic Rate Interface)**: 가입자용
  - 속도: 2B + D = 2×64kbps + 16kbps = 144kbps
- **PRI (Primary Rate Interface)**: 사업자용
  - 속도: 23B + D = 23×64kbps + 64kbps = 1.544Mbps (더 빠름!)

> 💡 **기억법**: BRI는 Basic(기본), PRI는 Primary(주요) → PRI가 더 빠름!

## 3. 고속 광역 통신망 (MAN)

**MAN (Metropolitan Area Network)**: 도시 규모의 고속 네트워크

### 3.1 DQDB (Distributed Queue Dual Bus)
- MAN의 **표준 프로토콜** (IEEE 802.6)
- **이중 버스** 구조: 양방향 전송 가능

### 3.2 프레임 릴레이 (Frame Relay)
- **패킷을 빠르게 중계**하는 기술
- X.25보다 간단하고 빠름 (오류 제어 최소화)

### 3.3 SMDS (Switched Multimegabit Data Service)
- DQDB의 **고속 버전**
- 수 Mbps급 속도

### 3.4 B-ISDN (Broadband ISDN)
**광대역 종합 정보 통신망**

> ⚠️ **중요한 개념**: 단순히 N-ISDN의 속도만 높인 게 **아님**!

- 기존 네트워크와 **완전히 새롭게 구축**한 통신망
- ATM 기술 기반
- 멀티미디어 서비스 지원

### 3.5 ATM (Asynchronous Transfer Mode)
**비동기 전송 모드**

**특징**:
- **53바이트 고정 크기 셀** 사용
  - 헤더: 5바이트
  - 데이터: 48바이트
- 고속 전송 (수백 Mbps ~ Gbps)
- B-ISDN의 핵심 기술

> 💡 **시험 꿀팁**: ATM = 53바이트 셀! 이거 자주 나옴

### 3.6 캐리어 이더넷 (Carrier Ethernet)
- IEEE 802.XX의 **고속 버전**
- WAN과 MAN에 **적합**하게 확장
- 기존 이더넷 기술을 광역망에 적용

## 4. 광대역 융합망

### 4.1 BCN (Broadband Convergence Network)
**광대역 융합망**

**개념**: 음성, 데이터, 영상 등 **각종 서비스를 통합**한 네트워크

**핵심 특징**:
- **개방형 플랫폼** (Open Platform)
- **OpenAPI**: Application Program Interface를 개방
  - 다양한 서비스 개발자들이 자유롭게 활용 가능

> 🎯 **비유**: 스마트폰처럼 여러 앱(서비스)을 자유롭게 설치해서 쓸 수 있는 네트워크!

### 4.2 UBcN (Ultra Broadband convergence Network)
**초광대역 융합망**

- BCN보다 **10배 빠른** 속도
- 초고화질 영상, 클라우드 서비스 등 대용량 서비스 지원

### 4.3 기타 융합망 관련 개념

#### VAN (Value Added Network)
- **부가가치 통신망**
- 기본 통신망에 추가 서비스를 제공
- 예: 전자상거래, EDI(전자문서교환)

#### MVNO (Mobile Virtual Network Operator)
- **알뜰폰** 사업자
- 자체 통신망 없이 기간통신사의 망을 빌려서 서비스

#### USN (Ubiquitous Sensor Network)
- **유비쿼터스 센서 네트워크**
- **언제 어디서든** 통신 가능
- IoT의 기반 기술

#### 국가정보통신망
- 정부 및 공공기관의 정보통신망

## 5. 초고속 유선 가입자망

**개념**: 통신사에서 우리 집까지 연결되는 **마지막 구간** (Last Mile)

### 5.1 DSL (Digital Subscriber Line) 계열

**DSL 스펙트럼 구조**:

```
주파수 →
[음성][상향링크(Upstream)][하향링크(Downstream)]
  ↑         ↑                      ↑
전화용  우리가 보내는 것      받는 것(다운로드)
```

> 💡 **핵심**: 하향링크가 상향링크보다 **훨씬 넓음**! (우리는 보는 게 더 많으니까)

#### 📌 ADSL (Asymmetric DSL)
- **비대칭** DSL
- 하향 속도 > 상향 속도
- **가장 많이 사용**하는 방식

#### 📌 SDSL (Symmetric DSL)
- **대칭** DSL
- 하향 속도 = 상향 속도
- 기업용 (업로드도 많이 하는 경우)

#### 📌 UDSL (Universal DSL)
- **전화국** 전용
- 장거리 전송용

> 💡 **기억법**: A는 Asymmetric(비대칭), S는 Symmetric(대칭), U는 Universal(전화국)

---

### 5.2 FTTx (Fiber To The x)

**개념**: 광케이블을 **어디까지** 연결하느냐에 따른 분류

#### 📌 FTTH (Fiber To The Home)
- 광케이블이 **집 안까지** (최고 품질)
- 가장 빠르지만 비용이 높음

#### 📌 FTTB (Fiber To The Building)
- 광케이블이 **건물까지**
- 건물 내부는 기존 선 사용

#### 📌 FTTC (Fiber To The Curb/Cabinet)
- 광케이블이 **도로 옆 박스까지**
- 집까지는 전화선 사용

#### 📌 FTTO (Fiber To The Office)
- 광케이블이 **사무실까지**

> 🎯 **속도 순서**: FTTH > FTTB > FTTO > FTTC

---

### 5.3 HFC (Hybrid Fiber Coax)

**특징**:
- **광케이블 + 동축케이블** 혼합
- 케이블 TV 방송망 활용
- 광케이블로 일정 구간까지 → 동축케이블로 각 가정까지

**장점**:
- 기존 케이블 TV 인프라 활용 가능
- 비용 효율적

**단점**:
- 동축케이블 구간에서 속도 저하
- FTTH보다 품질이 낮음

---

## 📝 시험 대비 핵심 요약

### ⭐ 꼭 외워야 할 것들

1. **WAN은 OSI 1~3계층** 커버
2. **축적 교환은 실시간 전송 안 됨**
3. **패킷 헤더에는 무조건 주소 포함**
4. **가상회선**: 경로 미리 설정 vs **데이터그램**: 독립적 전송
5. **N-ISDN**: BRI(가입자) vs PRI(사업자, 더 빠름)
6. **B-ISDN**: 단순 속도 증가 아님, 완전 새 통신망
7. **ATM**: 53바이트 고정 셀
8. **UBcN**: BCN보다 10배 빠름
9. **ADSL**: 하향 > 상향 (가장 많이 씀)
10. **FTTH**: 집까지 광케이블 (최고 품질)

### 🎯 문제 풀이 꿀팁

1. **교환 방식** 묻는 문제: 회선 vs 축적 구분 먼저!
2. **패킷 교환** 문제: 가상회선(경로 고정) vs 데이터그램(독립적) 특징 파악
3. **다중화** 문제: TDM(시간) vs FDM(주파수) 헷갈리지 말기
4. **DSL 종류**: A(비대칭), S(대칭), U(전화국) 기억!
5. **FTTx 속도**: H(Home) > B(Building) > O(Office) > C(Curb)

### ⚠️ 자주 틀리는 부분

- **B-ISDN ≠ N-ISDN의 고속 버전** (완전 새로운 망!)
- **축적 교환**: 실시간 X (이거 함정 조심)
- **데이터그램**: 순서 정보 포함! (순서 바뀔 수 있어서)
- **NT**: ISDN 전용 모뎀 (일반 모뎀 아님)
- **SMDS**: DQDB의 고속 버전 (별개 기술 아님)
