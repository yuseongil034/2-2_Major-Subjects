## 1. 정보통신망의 기본 개념

### 🔌 통신망이란?
여러 대의 컴퓨터나 통신장비를 **서로 연결**해서 데이터를 주고받을 수 있게 만든 시스템이에요.

### 📦 정보통신망의 3대 구성요소

| 구성요소 | 역할 | 예시 |
|---------|------|------|
| **단말기** | 데이터를 보내고 받는 장치 | 컴퓨터, 스마트폰 |
| **통신회선** | 데이터가 지나가는 길 | UTP 케이블, 광케이블 |
| **교환기** | 데이터를 올바른 목적지로 전달 | 스위치, 라우터 |

> 💡 **비유**: 우편 시스템을 생각해보세요!
> - 단말기 = 편지를 쓰고 받는 사람
> - 통신회선 = 도로
> - 교환기 = 우체국 (어디로 보낼지 결정)

## 2. 네트워크 분류

### 📏 범위와 연결 방식에 따른 분류

#### ⚡ LAN (Local Area Network) - 근거리 통신망
- **범위**: 가장 작음 (한 건물, 학교, 회사 내부)
- **속도**: 1~20 Mbps (빠름!)
- **특징**: 짧은 거리, 높은 신뢰성

#### 🌐 WAN (Wide Area Network) - 광역 통신망
- **범위**: 도시와 도시 연결 (10km 이상)
- **연결 방식**: 점대점 (Point-to-Point)
- **예시**: 서울-부산 연결

#### 🏙️ MAN (Metropolitan Area Network) - 도시권 통신망
- **범위**: 한 도시 전체 (LAN보다 크고 WAN보다 작음)
- **특징**: Metropolitan(대도시)의 약자

#### 💼 VAN (Value Added Network) - 부가가치 통신망
- **특징**: 기본 통신에 **추가 서비스**를 제공하는 네트워크
- **예시**: 결제 시스템, EDI(전자문서교환)

---

### 🔗 구성 형태에 따른 분류

#### 1️⃣ 망형 (Mesh)
```
    A --- B
    |  X  |
    C --- D
```
- **특징**: 모든 노드가 서로 연결됨
- **장점**: 효율성 최고, 한 선이 끊겨도 다른 경로 이용 가능
- **단점**: 💸 비용이 많이 듦 (선이 가장 많음)

#### 2️⃣ 링형 (Ring)
```
    A → B
    ↑   ↓
    D ← C
```
- **특징**: 원형으로 연결, 데이터가 한 방향으로 순환
- **장점**: 구조가 단순
- **단점**: 한 곳이 끊어지면 전체 영향

#### 3️⃣ 성형 (Star) ⭐
```
      중앙
    / | | \
   A  B C  D
```
- **특징**: 중앙 노드(허브/스위치)에 모두 연결
- **장점**: 관리가 쉬움, 노드 추가/제거 편함
- **단점**: ⚠️ **중앙 노드 고장 시 전체 다운!**
- **선의 개수**: 노드 개수만큼 필요

#### 4️⃣ 버스형 (Bus)
```
A---B---C---D
    (공통 선)
```
- **특징**: 하나의 공통 선에 모두 연결
- **장점**: 선이 적게 필요, 설치 간단
- **단점**: 공통 선 고장 시 전체 영향

#### 5️⃣ 트리형 (Tree)
```
        중앙
       /   \
      B     C
     / \   / \
    D  E  F  G
```
- **특징**: 계층적 구조 (성형 + 버스형의 조합)
- **장점**: 확장성 좋음
- **단점**: 상위 노드 고장 시 하위 전체 영향

## 3. 교환 방식

### 📞 회선 교환 (Circuit Switching)
- **방식**: 통신 전에 **전용 회선을 먼저 연결**
- **특징**: 
  - 연결 후에는 독점적으로 사용
  - 실시간 통신에 적합
- **예시**: 전화 통화 ☎️
- **장점**: 지연 시간 일정, 품질 보장
- **단점**: 회선을 사용 안 해도 독점하므로 비효율적

> 💡 **비유**: 친구와 통화할 때, 전화선을 연결한 후 대화하는 것

### 📬 축적 교환 (Store-and-Forward Switching)

#### 1️⃣ 메시지 교환
- **방식**: 메시지 전체를 저장했다가 한 번에 전송
- **특징**: 메시지 단위로 처리

#### 2️⃣ 패킷 교환 ⭐ (중요!)
- **방식**: 데이터를 **작은 패킷**으로 나눠서 전송
- **특징**: 
  - 각 패킷이 독립적으로 이동
  - 목적지에서 다시 조립
- **장점**: 효율적, 회선 공유 가능
- **예시**: 인터넷 📧

> 💡 **비유**: 큰 짐을 여러 택배 상자로 나눠서 보내는 것

## 4. 정보통신망 발전 과정

```
단독 시스템 → ARPANET → 인터넷 → ISDN → 인터네트워킹
```

### 📜 역사적 흐름

1. **단독 시스템**: 컴퓨터 1대 + 주변장치만 연결
2. **ARPANET**: 최초의 패킷 교환 네트워크 (인터넷의 시조)
3. **인터넷**: 전 세계를 연결하는 네트워크
4. **ISDN**: 종합정보통신망 (음성, 데이터 통합)
5. **인터네트워킹**: 여러 종류의 네트워크를 **통합 연결**

## 5. 근거리 통신망 (LAN)

### ⚡ LAN의 특징 (시험 출제!)

| 특징 | 설명 |
|------|------|
| **고속 전송** | 짧은 거리 → 빠른 속도 |
| **짧은 경로** | 전송 지연 최소화 |
| **높은 신뢰성** | 오류율 낮음 |
| **빠른 행정처리** | 관리가 효율적 |
| **통합 정보망** | 다양한 서비스 통합 |

### 🔧 LAN 구성 요소

#### 하드웨어
- **서버**: 자원을 제공하는 컴퓨터
- **클라이언트**: 서버의 자원을 사용하는 컴퓨터
- **전송 매체**: 데이터가 지나가는 물리적 경로

#### 전송 매체의 종류

| 종류 | 특징 |
|------|------|
| **UTP** (Unshielded Twisted Pair) | 비차폐 연선, 가장 흔함 |
| **STP** (Shielded Twisted Pair) | 차폐 연선, 간섭 방지 |
| **FTP** (Foiled Twisted Pair) | 포일 차폐 |
| **동축 케이블** | 중심 도체 + 외부 차폐 |
| **광케이블** | 빛으로 데이터 전송, 가장 빠름 💨 |

#### 소프트웨어
- **네트워크 운영체제** (NOS)
- **프로토콜**: 통신 규약

## 6. 네트워크 연결 장비와 OSI 계층

### 🔌 주요 네트워크 장비

#### 📡 MODEM (변조/복조 장치)
- **역할**: 디지털 신호 ↔ 아날로그 신호 변환
- **필요성**: 전화선(아날로그)으로 디지털 데이터 전송

#### 🔄 리피터 (Repeater)
- **역할**: 신호 증폭 및 재생
- **목적**: 먼 거리 전송 시 약해진 신호 복원

#### 🌐 허브 (Hub)
1. **더미 허브** (Dumb Hub)
   - 받은 신호를 모든 포트로 전송 (브로드캐스트)
   - 충돌 발생 가능
   
2. **스위칭 허브** (Switching Hub)
   - 목적지를 판단해서 해당 포트로만 전송
   - 충돌 없음, 효율적

#### 🌉 브릿지 (Bridge)
- **역할**: 두 개의 LAN을 연결
- **기능**: MAC 주소 기반 필터링

#### 🛤️ 라우터 (Router) ⭐
- **역할**: **내부 네트워크와 외부 네트워크 연결**
- **기능**: IP 주소 기반 경로 설정
- **중요도**: 매우 중요! (인터넷 연결의 핵심)

#### 📡 트랜시버 (Transceiver)
- **사용 조건**: 전송 매체가 **동축 케이블**일 때
- **역할**: 송신기 + 수신기 기능

#### 🔧 DSU (Digital Service Unit)
- **역할**: 디지털 회선 연결 장치

#### 🚪 게이트웨이 (Gateway)
- **역할**: 서로 다른 프로토콜을 사용하는 네트워크 연결
- **특징**: 프로토콜 변환 기능

---

### 🎯 OSI 7계층별 장비 매칭 (시험 필수! 🔥)

| OSI 계층 | 계층 번호 | 해당 장비 |
|---------|----------|---------|
| **물리 계층** (Physical) | 1계층 | 리피터, 허브, 트랜시버 |
| **데이터링크 계층** (Data Link) | 2계층 | 브릿지, 스위치 |
| **네트워크 계층** (Network) | 3계층 | 라우터 |
| **전송 계층** (Transport) | 4계층 | - |
| **세션 계층** (Session) | 5계층 | - |
| **표현 계층** (Presentation) | 6계층 | - |
| **응용 계층** (Application) | 7계층 | 게이트웨이 |

> ⚠️ **암기 팁**: "리허트 / 브스 / 라 / - / - / - / 게"
> - 1계층: **리**피터, **허**브, **트**랜시버
> - 2계층: **브**릿지, **스**위치
> - 3계층: **라**우터
> - 7계층: **게**이트웨이

## 7. LAN 접근 방식

### 🔄 토큰 방식의 원리

#### 1️⃣ 토큰 링 (Token Ring)

**동작 원리**:
```
자유 토큰 순환 → A가 데이터 전송하려면 → 자유 토큰 획득 
→ 사용중 토큰으로 변경 → 목적지 주소 + 데이터 첨부 
→ 링을 따라 순환 → D가 데이터 수신 → 다시 A로 돌아옴 
→ 자유 토큰으로 변경
```

**특징**:
- 토큰이 **계속 순환**하면서 데이터 전송 권한 부여
- **충돌 없음** (한 번에 하나만 전송)
- 송신자 주소 + 목적지 주소 + 데이터 포함

> 💡 **비유**: 말하기 권한을 나타내는 마이크가 계속 돌아가는 회의실

#### 2️⃣ 토큰 버스 (Token Bus)

**특징**:
- 버스 형태지만 토큰 사용
- **충돌 가능성 있음** → **CSMA/CD와 병행** 사용
- 수신 확인 신호 포함

---

### ⚡ CSMA/CD (충돌 감지 프로토콜) ⭐

**CSMA/CD**: Carrier Sense Multiple Access with Collision Detection

**동작 원리**:
1. 전송 전에 회선이 **비어있는지 확인** (Carrier Sense)
2. 비어있으면 데이터 전송
3. **충돌 발생 시**:
   - 즉시 전송 중단
   - 대기 시간 후 재전송
   - 각 장치는 **랜덤한 시간** 대기 (충돌 방지)

> 💡 **비유**: 여러 사람이 동시에 말하려다가 겹치면, 잠깐 쉬었다가 다시 말하는 것

**사용 환경**:
- 이더넷(Ethernet)에서 주로 사용
- 토큰 버스와 함께 사용

## 8. 무선 LAN (IEEE 802.11)

### 📡 IEEE 802 표준

- **유선 LAN**: IEEE 802.XX
- **무선 LAN**: IEEE 802.11 ⭐

### 🌐 IEEE 802.11 프로토콜 세부 표준 (시험 필수! 🔥)

| 표준 | 발표 시기 | 주파수 대역 | 대역폭 | 변조 방식 | 안테나 기술 | 스트림당 전송속도 | 최대 전송속도 |
|------|----------|------------|--------|----------|-----------|---------------|-------------|
| **802.11** | 1997년 | 2.4 GHz | 20 MHz | FHSS/DSSS | SISO | - | 2 Mbps |
| **802.11a** | 1999년 | 5 GHz | 20 MHz | OFDM | SISO | 54 Mbps | 54 Mbps |
| **802.11b** | 1999년 | 2.4 GHz | 20 MHz | DSSS | SISO | 11 Mbps | 11 Mbps |
| **802.11g** | 2003년 | 2.4 GHz | 20 MHz | OFDM | SISO | 54 Mbps | 54 Mbps |
| **802.11n** | 2009년 | 2.4/5 GHz | 20/40 MHz | MIMO-OFDM | MIMO (4×4) | 150 Mbps | 600 Mbps |
| **802.11ac** | 2013년 | 5 GHz | 20/40/80/160 MHz | MIMO-OFDM | MU-MIMO (8×8) | 866.7 Mbps | 6.9 Gbps |
| **802.11ad** | 2012년 | 60 GHz | 2160 MHz | SC/OFDM | SISO | 7 Gbps | 7 Gbps |
| **802.11ah** | 2017년 | 900 MHz | 1/2/4/8/16 MHz | OFDM | SISO/MIMO | - | 347 Mbps |
| **802.11ay** | 2021년 | 60 GHz | 8640 MHz | SC/OFDM | MIMO | - | 20~40 Gbps |

---

### 📝 변조 방식 설명

| 변조 방식 | 설명 |
|---------|------|
| **DSSS** (Direct Sequence Spread Spectrum) | 직접 시퀀스 확산 스펙트럼 |
| **OFDM** (Orthogonal Frequency Division Multiplexing) | 직교 주파수 분할 다중화 |
| **MIMO** (Multiple Input Multiple Output) | 다중 안테나 기술 |
| **SC** (Single Carrier) | 단일 반송파 |

---

### 🛡️ CSMA/CA (충돌 회피)

**CSMA/CA**: Carrier Sense Multiple Access with Collision **Avoidance**

**차이점**:
- 유선: CSMA/**CD** (Collision **Detection** - 감지)
- 무선: CSMA/**CA** (Collision **Avoidance** - 회피)

**이유**: 무선은 충돌 감지가 어려우므로 **사전에 회피**

**동작 원리**:
1. 전송 전에 회선 확인
2. RTS (Request to Send) 신호 전송
3. CTS (Clear to Send) 응답 대기
4. 응답 받으면 데이터 전송

## 9. IoT와 OSI 계층

### 🌐 IoT의 OSI 계층 사용 특징 ⭐

IoT는 OSI 7계층 중에서:
- ✅ **1, 2, 3, 4, 7 계층 사용**
- ❌ **5, 6 계층 미사용**

**이유**:
- **5계층 (세션)**: 연결 관리가 단순해서 불필요
- **6계층 (표현)**: 데이터 형식 변환이 응용 계층에서 처리됨

**IoT 통신 스택 구조**:
```
7계층: 응용 (CoAP, MQTT 등)
4계층: 전송 (TCP/UDP)
3계층: 네트워크 (IPv6, 6LoWPAN)
2계층: 데이터링크
1계층: 물리
```

> 💡 **핵심**: IoT는 경량화를 위해 꼭 필요한 계층만 사용!

### 🔥 반드시 암기할 것!

1. **네트워크 분류**: LAN, WAN, MAN, VAN의 범위와 특징
2. **토폴로지**: 성형(중앙 고장 시 전체 다운), 망형(비용 많음)
3. **OSI 계층별 장비**: 리피터/허브(1), 브릿지(2), 라우터(3), 게이트웨이(7)
4. **802.11 표준**: 주파수, 속도, 변조 방식 (표 전체!)
5. **CSMA/CD vs CSMA/CA**: 유선은 Detection, 무선은 Avoidance
6. **IoT**: 5, 6계층 미사용

### 💡 시험 빈출 포인트

- 라우터 = 내부/외부 네트워크 연결
- 트랜시버 = 동축 케이블 사용 시
- 토큰 링 = 충돌 없음, 순환 방식
- 802.11n부터 MIMO 기술 사용
- LAN 특징: 고속, 짧은 경로, 높은 신뢰성
